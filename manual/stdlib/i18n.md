# aivi.i18n

Internationalisation helpers for user-facing text.

This module is deliberately small right now:

- `tr` returns the text you pass in unchanged
- `trn` chooses the singular text when the count is `1`, otherwise the plural text

In other words, you can start marking strings for translation today, even though a full
locale and message-catalog system is not wired in yet.

## Import

```aivi
use aivi.i18n (
    tr
    trn
    translate
    translatePlural
    t_
)
```

## Overview

| Function | Type | Description |
|----------|------|-------------|
| `tr` | `Text -> Text` | Mark a single string for translation |
| `trn` | `Text -> Text -> Int -> Text` | Choose singular or plural text |
| `translate` | `Text -> Text` | Longer alias for `tr` |
| `translatePlural` | `Text -> Text -> Int -> Text` | Longer alias for `trn` |
| `t_` | `Text -> Text` | Short alias for `tr` |

## Functions

### tr

```aivi
tr : Text -> Text
```

Mark a single string for translation.

Current behaviour: the runtime returns the same text unchanged.

```aivi
use aivi.i18n (tr)

value saveLabel : Text =
    tr "Save"
```

### trn

```aivi
trn : Text -> Text -> Int -> Text
```

Choose between a singular and plural form.

Current behaviour: when the count is `1`, `trn` returns the first string. For any other
count, it returns the second string.

`trn` does not automatically insert the number for you. If you want the count in the text,
include it in the string yourself.

```aivi
use aivi.i18n (trn)

type Int -> Text
func fileCountLabel = count =>
    trn "1 file" "{count} files" count
```

### translate

```aivi
translate : Text -> Text
```

Friendly alias for `tr`. Use it when a longer name reads better in your code.

### translatePlural

```aivi
translatePlural : Text -> Text -> Int -> Text
```

Friendly alias for `trn`.

### t_

```aivi
t_ : Text -> Text
```

Short alias for `tr`. This can be handy in UI code where translated labels appear often.

```aivi
use aivi.i18n (t_)

value openLabel : Text =
    t_ "Open"
```

## Example — mark UI text early

```aivi
use aivi.i18n (
    tr
    trn
)

value title : Text = tr "Downloads"

type Int -> Text
func itemLabel = count =>
    trn "1 item" "{count} items" count
```

Even with today's simple runtime behaviour, this makes it obvious which strings are intended
for future translation work.
